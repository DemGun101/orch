# 4.3 — Error Handler & Wire Resilience Into Core

```
Continuing "cowork-orchestrator". Rate limiter and checkpointing are done. Now implement centralized error handling and wire everything together. The project uses the OpenAI SDK with Gemini (not Anthropic).

## 1. src/resilience/error-handler.ts — Centralized Error Handling

### Custom Error Hierarchy
- `OrchestratorError extends Error` — base: { code: string, context: Record<string,unknown>, recoverable: boolean }
- `TaskExecutionError extends OrchestratorError` — code='TASK_EXECUTION', recoverable=true
- `AgentError extends OrchestratorError` — code='AGENT_ERROR', recoverable=true
- `ToolExecutionError extends OrchestratorError` — code='TOOL_EXECUTION', recoverable=true
- `APIError extends OrchestratorError` — { statusCode: number }. recoverable=true for 429/500/503, false for 401/403
- `WorkflowError extends OrchestratorError` — code='WORKFLOW_ERROR'
- `ValidationError extends OrchestratorError` — code='VALIDATION', recoverable=false
- `TimeoutError extends OrchestratorError` — code='TIMEOUT', recoverable=true

Note: Gemini uses 429 (rate limit) and 503 (service unavailable) instead of Anthropic's 529.

### ErrorRecoveryAction type
type ErrorRecoveryAction = 'RETRY' | 'REASSIGN' | 'DECOMPOSE' | 'SKIP' | 'ESCALATE' | 'ABORT';

### ErrorHandler class
- Constructor takes optional config: { maxRetries: 3, escalationThreshold: 5 }
- Internal: error history per agent/task for pattern detection

- `handleError(error: OrchestratorError): Promise<ErrorRecoveryAction>`
  Decision matrix:
  - APIError 429/503 + retryCount < max → RETRY
  - APIError 500 + retryCount < max → RETRY
  - APIError 401/403 → ABORT
  - TaskExecutionError + retryCount < max → RETRY
  - TaskExecutionError + retryCount >= max → REASSIGN
  - AgentError (repeated from same agent 3+ times) → REASSIGN
  - ToolExecutionError → RETRY (once), then SKIP if non-critical
  - TimeoutError → RETRY with longer timeout, then DECOMPOSE
  - ValidationError → ABORT
  - Default: ESCALATE

- `withErrorHandling<T>(fn: () => Promise<T>, context: { taskId?, agentId?, operation: string }): Promise<T>`
  - Wraps fn in try/catch, converts errors to OrchestratorError, calls handleError

- `recordError(agentId: string, error: OrchestratorError)` — track for pattern detection
- `getErrorPatterns(agentId: string): { totalErrors, recentErrors, isDegraded }`

- Emit events: 'error:occurred', 'error:recovered', 'error:escalated'

## 2. Wire Resilience Into OrchestrationEngine

Update `src/core/orchestrator.ts`:
- Initialize: ErrorHandler, APIRateLimiter, ConcurrencyLimiter, CheckpointManager
- Create a per-agent CircuitBreaker Map
- Wrap `submitTask()`:
  - Acquire rate limit before API calls
  - Use circuit breaker per agent
  - On error: consult ErrorHandler, execute recovery action (retry, reassign, etc.)
- Graceful shutdown on SIGTERM/SIGINT:
  1. Set isShuttingDown flag — stop accepting new tasks
  2. Wait for running tasks with 30s timeout
  3. Checkpoint all active workflows
  4. Close DB
  5. process.exit(0)

## 3. Wire Resilience Into LLMAgent (formerly ClaudeAgent)

Update `src/agents/llm-agent.ts`:
- Accept optional ErrorHandler, RateLimiter, CircuitBreaker in constructor (or via setters)
- In `execute()`: wrap API call with rate limiter → circuit breaker → retry strategy
- On 429: call rateLimiter.handleRateLimitResponse() with retry-after value from error headers
- Track error rate in stats

## 4. Wire Resilience Into WorkflowEngine

Update `src/core/workflow-engine.ts`:
- Accept CheckpointManager in constructor
- After each node completes: create checkpoint
- In execute(): check for existing checkpoint, if found use restoreFromCheckpoint() to skip already-done nodes
- On node failure: consult ErrorHandler for recovery action
- Add workflow-level timeout (default from config.defaultTimeout * number of nodes)

## 5. Wire Resilience Into ToolExecutor

Update `src/tools/tool-executor.ts`:
- Create a CircuitBreaker per tool (Map<string, CircuitBreaker>)
- Wrap each tool execution: circuit breaker → timeout → error handling
- If a tool's circuit breaker is OPEN, return ToolResult with error "Tool temporarily unavailable"

Run `npx tsc --noEmit`. Fix ALL issues.
```
