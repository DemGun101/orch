# 3.2 — MCP Protocol Client & Wire Tool Agent

```
Continuing "cowork-orchestrator". Tool registry and executor are done. Now implement MCP client and wire real tool execution into ToolAgent. The project uses the OpenAI SDK with Gemini — tool calls come as message.tool_calls with function.name and function.arguments.

## 1. src/tools/mcp-client.ts — MCP Protocol Client

Implement `MCPClient` class for connecting to MCP (Model Context Protocol) servers:

- Constructor takes optional ToolRegistry to auto-register discovered tools

- `connect(serverConfig: MCPServerConfig): Promise<void>`
  - MCPServerConfig: { command: string, args?: string[], env?: Record<string,string> }
  - Spawn a child process with the given command
  - Communicate via stdin/stdout using JSON-RPC 2.0
  - Send `initialize` request: { jsonrpc: '2.0', id: 1, method: 'initialize', params: { protocolVersion: '2024-11-05', capabilities: {} } }
  - Wait for initialize response, store server capabilities
  - Send `initialized` notification

- `disconnect(): Promise<void>` — kill the child process cleanly

- `listTools(): Promise<ToolDefinition[]>` — send `tools/list` request, convert response to ToolDefinition[]

- `callTool(name: string, args: unknown): Promise<unknown>` — send `tools/call` request with { name, arguments: args }, return the result content

- `listResources(): Promise<Resource[]>` — send `resources/list` request

- `readResource(uri: string): Promise<ResourceContent>` — send `resources/read` request

- Internal JSON-RPC transport:
  - `sendRequest(method, params): Promise<any>` — assigns incrementing ID, writes to stdin, waits for matching response on stdout
  - `sendNotification(method, params)` — no ID, no response expected
  - Parse stdout line-by-line, match responses by ID
  - Handle server notifications
  - Timeout: 30s per request

- Auto-register: after connect, call listTools() and register all into ToolRegistry (if provided)
- Health check: `isConnected(): boolean`

## 2. Update src/agents/tool-agent.ts — Wire Real Tool Execution

Update ToolAgent to use real ToolExecutor instead of stubs:
- Add `setToolExecutor(executor: ToolExecutor)` method
- In the tool execution loop, when Gemini returns tool_calls (OpenAI format):
  - Each tool_call has: { id, type: 'function', function: { name, arguments } }
  - JSON.parse the arguments string
  - Call `toolExecutor.execute(toolName, parsedArgs)` for each tool call
  - If tool is in requireApproval list: emit event, wait for approval
  - Send ToolResult back as role: 'tool' messages: { role: 'tool', tool_call_id: call.id, content: JSON.stringify(result) }
  - Handle tool errors: send error message back so the LLM can adapt
- Track tool usage per agent: { toolName, callCount, errorCount, avgDuration }
- Add `getToolUsageStats()` method

Run `npx tsc --noEmit`. Fix any issues.
```
