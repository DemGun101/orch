# 2.1 — AI Task Decomposer

```
Continuing "cowork-orchestrator". Phase 1 (foundation) is complete. Now implement the first intelligence module. Import the Anthropic SDK. Use `claude-sonnet-4-20250514` as the default model.

## src/intelligence/task-decomposer.ts — AI Task Decomposition

Implement `TaskDecomposer` class:
- Constructor takes `Anthropic` client instance and optional config `{ model?: string, maxSubtasks?: number }` (defaults: model='claude-sonnet-4-20250514', maxSubtasks=10)

- `decompose(task: Task, availableCapabilities: AgentCapability[]): Promise<DecompositionResult>`
  - Builds a system prompt: "You are a task decomposition expert. Break the given task into subtasks that can each be handled by one of the available agent capabilities. Return structured JSON."
  - Sends the task description + list of available capabilities (name + description) to Claude
  - Uses tool_use with a Zod-validated decomposition schema to get structured output:
    {
      subtasks: [{ name: string, description: string, requiredCapability: string, priority: TaskPriority, estimatedComplexity: string, dependencies: string[] }],
      reasoning: string,
      parallelGroups: string[][]
    }
  - Validates response with Zod, converts to Task objects, sets up dependency links between subtasks
  - Edge cases: if task is too simple (Claude says so), return the original task unchanged. If too complex (>maxSubtasks), recursively decompose the largest subtasks.

- `estimateComplexity(task: Task): Promise<'trivial' | 'simple' | 'moderate' | 'complex' | 'epic'>`
  - Quick Claude call with short prompt: "Rate the complexity of this task on a scale: trivial, simple, moderate, complex, epic. Task: {task.description}. Reply with just the rating."
  - Parse and return

- `suggestWorkflow(task: Task, capabilities: AgentCapability[]): Promise<Workflow>`
  - Claude designs a workflow structure: nodes (what to do) + edges (dependencies)
  - Code converts Claude's response to proper Workflow/WorkflowNode/WorkflowEdge objects with UUIDs

Define the `DecompositionResult` type in this file or in types.ts:
  interface DecompositionResult {
    subtasks: Task[];
    reasoning: string;
    parallelGroups: string[][];
  }

Run `npx tsc --noEmit` to verify it compiles.
```
