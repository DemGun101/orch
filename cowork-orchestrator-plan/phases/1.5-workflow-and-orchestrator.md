## 1. src/core/workflow-engine.ts — WorkflowEngine

Implement `WorkflowEngine` class:
- Constructor takes TaskManager, AgentRegistry, and optional PersistenceLayer
- Internal `activeWorkflows: Map<string, { workflow: Workflow, status, nodeStatuses }>`
- `execute(workflow: Workflow): Promise<WorkflowResult>`
  - Validates the DAG has no cycles (use DFS-based cycle detection)
  - Computes topological sort to determine execution order
  - Executes nodes respecting the DAG:
    - Find all nodes with no unresolved incoming edges → run them in parallel (Promise.all)
    - When a node completes, pass its output into the workflow context
    - Check which new nodes now have all dependencies satisfied → run those
    - Handle conditional edges: if edge has a `condition`, evaluate it as a simple expression against workflow.context (use a basic evaluator or JSON path check)
    - Repeat until all nodes complete or one fails
  - Returns WorkflowResult with all outputs, timing, node counts
- `pause(workflowId: string)` — sets a flag that prevents new nodes from starting
- `resume(workflowId: string)` — clears the pause flag, resumes execution
- `getStatus(workflowId: string)` — returns current state of all nodes
- `cancel(workflowId: string)` — cancels all pending/running tasks in the workflow
- Checkpointing: after each node completes, save state to persistence (workflow context + node statuses + outputs)
- Helper: `validateDAG(nodes, edges): { valid: boolean, error?: string }` — exported for testing
- Helper: `topologicalSort(nodes, edges): string[]` — exported for testing

## 2. src/core/orchestrator.ts — Main OrchestrationEngine

Implement the top-level `OrchestrationEngine` class that wires everything together:
- Constructor takes optional `Partial<OrchestratorConfig>` (merges with DEFAULT_CONFIG)
- Initializes all subsystems:
  - PersistenceLayer (if persistence.enabled)
  - TaskManager
  - AgentRegistry
  - WorkflowEngine
  - MessageBus
  - SharedMemoryStore
  - ConversationHistory
- `start(): Promise<void>` — initializes persistence, starts the main processing loop
- `stop(): Promise<void>` — graceful shutdown: wait for running tasks (with 10s timeout), close DB
- `registerAgent(config: AgentConfig): BaseAgent` — creates a ClaudeAgent (or ToolAgent if config has tools), registers it, returns it
- `submitTask(input: Partial<Task>): Promise<TaskResult>` — creates task, finds best agent via registry, assigns task, waits for result
- `executeWorkflow(workflow: Workflow): Promise<WorkflowResult>` — delegates to WorkflowEngine
- Main loop (runs on setInterval, every 1 second):
  1. Get next task from TaskManager queue
  2. Find available agent via AgentRegistry.findBestMatch
  3. If match found, assign and execute (don't await — fire and forget, track via task status)
  4. Check running tasks for timeouts (task.timeout exceeded → fail the task)
  5. Process completed tasks: call TaskManager.onTaskComplete for parent task resolution
- `getStatus()` — returns { isRunning, activeAgents, pendingTasks, runningTasks, completedTasks }
- `getMetrics()` — aggregated metrics from all subsystems
