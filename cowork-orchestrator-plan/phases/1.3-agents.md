Continuing "cowork-orchestrator". Memory and communication layers are done. Now implement all agent classes. After each file, run `npx tsc --noEmit`.

## 1. src/agents/base-agent.ts — Abstract BaseAgent

Implement abstract `BaseAgent` class:
- Constructor takes `AgentConfig` (from src/core/types.ts)
- Properties: `id`, `name`, `role`, `status: AgentStatus` ('idle' | 'busy' | 'error' | 'offline'), `currentTasks: Map<string, Task>`
- Abstract method: `execute(task: Task): Promise<TaskResult>`
- Concrete methods:
  - `canHandle(task: Task): boolean` — checks if any capability name matches task.name or task description keywords
  - `getLoad(): number` — returns currentTasks.size / config.maxConcurrentTasks (0 to 1)
  - `assignTask(task: Task)` — adds to currentTasks, sets status to 'busy' if at capacity
  - `completeTask(taskId: string, result: TaskResult)` — removes from currentTasks, updates status
  - `failTask(taskId: string, error: string)` — removes from currentTasks, tracks error count
  - `onMessage(msg: AgentMessage)` — default message handler (can be overridden)
  - `getStats()` — returns { tasksCompleted, tasksFailed, avgExecutionTime, errorRate }
- Optional integration with MessageBus (via `setMessageBus(bus)`) and ConversationHistory (via `setHistory(history)`)

## 2. src/agents/claude-agent.ts — ClaudeAgent

Implement `ClaudeAgent` extends `BaseAgent`:
- Constructor additionally takes an Anthropic client instance (from @anthropic-ai/sdk)
- `execute(task: Task)` implementation:
  - Builds messages array from conversation history + task context
  - Calls `anthropic.messages.create()` with the agent's model, system prompt, temperature, maxTokens
  - Handles tool_use responses by returning tool name + inputs as stub (real execution in Phase 3)
  - Returns structured `TaskResult` with output, token usage (from response.usage), duration
  - Implements basic retry on transient errors (429, 500, 529) with simple delay
- `chat(userMessage: string): Promise<string>` — simple single-turn convenience method
- Token tracking: count input/output tokens from API response.usage
- Manages conversation history window: trim when token count approaches model context limit

## 3. src/agents/tool-agent.ts — ToolAgent

Implement `ToolAgent` extends `ClaudeAgent`:
- Has a list of `ToolDefinition[]` available tools
- Overrides `execute()` to include tools in the API call (using Claude's tool_use format)
- Implements a tool execution loop:
  1. Send message to Claude with tools
  2. If response has `tool_use` blocks, collect each tool call (stub: return { toolName, input } as string output)
  3. Send tool results back to Claude as tool_result messages
  4. Repeat until Claude returns a final `text` response or max iterations reached (10)
- `registerTool(tool: ToolDefinition)` / `removeTool(toolName: string)`

## 4. src/agents/human-in-the-loop.ts — HumanInTheLoopAgent

Implement `HumanInTheLoopAgent` extends `BaseAgent`:
- Used as an approval gate in workflows
- Has an internal `pendingApprovals: Map<string, { task, resolve, reject }>` map
- `execute(task: Task)`:
  - Emits an event `approval-needed` via eventemitter3 with task details
  - Returns a Promise that waits for `approve(taskId)` or `reject(taskId, reason)` to be called
  - Has a configurable timeout (default 5 minutes), auto-rejects on timeout with "Approval timed out"
- `approve(taskId: string): void` — resolves the pending promise with success TaskResult
- `reject(taskId: string, reason: string): void` — resolves with failed TaskResult
- `getPendingApprovals(): Task[]` — returns list of tasks awaiting approval

Run `npx tsc --noEmit` after all files. Fix any issues.
