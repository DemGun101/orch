Continuing "cowork-orchestrator". Retry strategies are done. Now implement rate limiting and checkpointing.

## 1. src/resilience/rate-limiter.ts — Rate Limiting

### RateLimiter class (Token Bucket algorithm)
- Config: { maxTokens: number, refillRate: number (tokens per second) }
- Internal: currentTokens (starts at maxTokens), lastRefill timestamp
- `acquire(tokens: number = 1): Promise<void>` — refill based on elapsed time, then: if enough tokens, consume and return. If not, calculate wait time, setTimeout, then consume.
- `tryAcquire(tokens: number = 1): boolean` — non-blocking. Refill, check, consume or return false.
- `getAvailable(): number` — current token count after refill

### APIRateLimiter extends with dual bucket
- Two internal RateLimiters: one for requests/minute, one for tokens/minute
- `acquireRequest(): Promise<void>` — acquire 1 from request bucket
- `acquireTokens(count: number): Promise<void>` — acquire from token bucket
- `recordUsage(inputTokens: number, outputTokens: number)` — for tracking actual usage
- `handleRateLimitResponse(retryAfterMs: number)` — on 429: pause the request bucket for retryAfterMs
- `getStats(): { requestsAvailable, tokensAvailable, requestsUsed, tokensUsed }`

### ConcurrencyLimiter class
- Config: { maxConcurrent: number }
- Internal: active count, waiting queue (array of resolve functions)
- `acquire(): Promise<() => void>` — if active < max, increment and return release function. Otherwise, push to queue and wait.
- Release function: decrements active, dequeues next waiter if any
- `getActive(): number`, `getWaiting(): number`

## 2. src/resilience/checkpointing.ts — Workflow Checkpoints

### CheckpointManager class
- Constructor takes PersistenceLayer

- `createCheckpoint(workflowId: string, state: WorkflowState): string`
  - WorkflowState: { nodeOutputs: Record<string, TaskResult>, nodeStatuses: Record<string, TaskStatus>, context: Record<string, unknown>, conversationSnapshots: Record<string, any[]>, pendingTasks: string[], timestamp: Date }
  - Generates checkpoint ID (UUID), serializes state, saves via persistence
  - Returns checkpoint ID

- `getCheckpoint(checkpointId: string): WorkflowState`
- `getLatestCheckpoint(workflowId: string): WorkflowState | undefined`

- `restoreFromCheckpoint(checkpointId: string): RestorationPlan`
  - RestorationPlan: { skipNodes: string[] (already completed), resumeNodes: string[] (were in-progress), rerunNodes: string[] (were pending), context: Record<string,unknown> }
  - Logic: completed nodes → skip, running/paused nodes → resume, pending → rerun

- `listCheckpoints(workflowId: string): CheckpointInfo[]`
  - CheckpointInfo: { id, workflowId, timestamp, nodeCount }

- `pruneCheckpoints(workflowId: string, keepLast: number)` — delete all but the N most recent
