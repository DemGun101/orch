# 1.4 — AgentRegistry & TaskManager

```
Continuing "cowork-orchestrator". All agent classes are done. Now implement the two core management classes. After each file, run `npx tsc --noEmit`.

## 1. src/core/agent-registry.ts — AgentRegistry

Implement `AgentRegistry` class:
- Internal `agents: Map<string, BaseAgent>`
- Optional PersistenceLayer for durability
- `register(agent: BaseAgent)` — adds agent to registry, optionally persists config
- `unregister(agentId: string)` — removes agent
- `get(agentId: string): BaseAgent | undefined`
- `findByCapability(capabilityName: string): BaseAgent[]` — find all agents that have a capability with matching name
- `findBestMatch(task: Task): BaseAgent | undefined` — score agents by:
  1. Capability match (does agent have a matching capability?) +10 points
  2. Current load (lower load = better) +5 * (1 - load) points
  3. Error rate (lower = better) -5 * errorRate points
  Pick the agent with highest score that is available
- `getAll(): BaseAgent[]`
- `getAvailable(): BaseAgent[]` — agents whose status is not 'offline' and getLoad() < 1
- `getMetrics()` — returns { totalAgents, availableAgents, avgLoad, totalTasksCompleted }
- Emit events via eventemitter3: 'agent:registered', 'agent:unregistered'

## 2. src/core/task-manager.ts — TaskManager

Implement `TaskManager` class:
- Internal tasks storage: `Map<string, Task>`
- Priority queue for pending tasks (sorted array — sort by priority rank: critical=0, high=1, medium=2, low=3, then by createdAt)
- Optional PersistenceLayer
- Emit events via eventemitter3
- Methods:
  - `createTask(input: Partial<Task>): Task` — generates UUID, sets defaults (status='pending', priority='medium', retryCount=0, maxRetries=3, timestamps), validates with Zod
  - `submitTask(task: Task)` — adds to queue, emits 'task:created'
  - `getNextTask(): Task | undefined` — returns highest priority task whose status is 'pending' AND all dependencies are in 'completed' status
  - `updateStatus(taskId: string, status: TaskStatus, output?: Record<string, unknown>)` — updates task, persists, emits 'task:{status}', updates timestamp
  - `getTask(taskId: string): Task`
  - `getSubtasks(parentId: string): Task[]` — all tasks with matching parentId
  - `decompose(taskId: string, subtasks: Partial<Task>[])` — creates subtask Task objects, sets parentId, adds original taskId's subtasks array, sets parent status to 'running'
  - `cancelTask(taskId: string)` — sets status to 'cancelled', cascading: also cancels all subtasks
  - `getTaskTree(taskId: string)` — returns task object with nested subtasks recursively
  - `onTaskComplete(taskId: string)` — checks if task has a parentId, if so checks if ALL sibling subtasks are 'completed', if yes marks parent as 'completed'
  - Dependency resolution: `areDependenciesMet(task: Task): boolean` — checks all task.dependencies are 'completed'

Run `npx tsc --noEmit` after both files. Fix any issues.
```
