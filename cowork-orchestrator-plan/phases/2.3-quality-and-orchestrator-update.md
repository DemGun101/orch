## 1. src/intelligence/quality-assessor.ts — AI Quality Assessment

Implement `QualityAssessor` class:
- Constructor takes an `OpenAI` client (from 'openai' package) and optional config `{ defaultThreshold?: number }` (default 70)

- `assess(task: Task, result: TaskResult): Promise<QualityReport>`
  - Sends task description + result output to Gemini via client.chat.completions.create()
  - Uses function calling for structured response:
    tools: [{ type: 'function', function: { name: 'quality_report', parameters: {
      type: 'object',
      properties: {
        overallScore: { type: 'number' },
        dimensions: { type: 'object', properties: { completeness: {type:'number'}, accuracy: {type:'number'}, coherence: {type:'number'}, relevance: {type:'number'} } },
        issues: { type: 'array', items: { type: 'object', properties: { severity: {type:'string'}, description: {type:'string'}, suggestion: {type:'string'} } } },
        passesThreshold: { type: 'boolean' },
        improvementSuggestions: { type: 'array', items: { type: 'string' } }
      }
    }}}]
    tool_choice: { type: 'function', function: { name: 'quality_report' } }
  - Parse response from: JSON.parse(response.choices[0].message.tool_calls[0].function.arguments)
  - Validate with Zod
  - Threshold is configurable per priority: critical=90, high=80, medium=70, low=50

- `compare(task: Task, results: TaskResult[]): Promise<ComparisonReport>`
  - Compare multiple outputs, rank them: { rankings: [{ taskResultIndex, score, reasoning }], bestIndex: number }

- `validateWorkflowOutput(workflow: Workflow, finalOutput: unknown): Promise<QualityReport>`
  - End-to-end check: does the workflow output satisfy the original workflow description?

- New constructor params or lazy initialization for: TaskDecomposer, AgentSelector, ConflictResolver, QualityAssessor (only created if OpenAI client / GEMINI_API_KEY is available)

  5. Quality-checks the final output
  6. Returns the result
