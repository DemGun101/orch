# 3.3 — Metrics, Audit Log & CLI Dashboard

```
Continuing "cowork-orchestrator". MCP tools are done. Now implement the monitoring layer.

## 1. src/monitoring/metrics.ts — Metrics Collection

Implement `MetricsCollector` class:
- Internal storage: Map<string, DataPoint[]> where DataPoint = { value: number, labels: Record<string,string>, timestamp: number }
- Config: { retentionMs: number (default 3600000 = 1hr), granularityMs: number (default 10000 = 10s) }

- `record(metric: string, value: number, labels?: Record<string, string>)` — stores a data point, auto-prunes old data beyond retention
- `increment(metric: string, labels?)` — shorthand for record(metric, current+1)
- `startTimer(metric: string, labels?): () => void` — captures start time, returns stop function that calls record() with elapsed ms
- `getMetrics(): MetricsSummary` — returns current snapshot of all metrics:
  For each metric: { name, current (latest value), min, max, avg, count, labels }
- `getTimeSeries(metric: string, duration: number): DataPoint[]` — data points within last `duration` ms
- `reset()` — clear all metrics

Built-in metric names (as constants):
  METRICS = {
    TASKS_TOTAL: 'tasks.total', TASKS_COMPLETED: 'tasks.completed', TASKS_FAILED: 'tasks.failed', TASKS_DURATION: 'tasks.duration_ms',
    AGENTS_ACTIVE: 'agents.active', AGENTS_UTILIZATION: 'agents.utilization',
    API_REQUESTS: 'api.requests', API_TOKENS_IN: 'api.tokens.input', API_TOKENS_OUT: 'api.tokens.output', API_LATENCY: 'api.latency_ms', API_ERRORS: 'api.errors',
    TOOLS_EXECUTIONS: 'tools.executions', TOOLS_ERRORS: 'tools.errors', TOOLS_DURATION: 'tools.duration_ms',
    WORKFLOW_EXECUTIONS: 'workflow.executions', WORKFLOW_DURATION: 'workflow.duration_ms',
    MESSAGES_SENT: 'messages.sent', MESSAGES_FAILED: 'messages.failed'
  }

## 2. src/monitoring/audit-log.ts — Audit Trail

Implement `AuditLogger` class:
- Constructor takes PersistenceLayer
- Event type constants: TASK_CREATED, TASK_ASSIGNED, TASK_COMPLETED, TASK_FAILED, AGENT_REGISTERED, TOOL_EXECUTED, WORKFLOW_STARTED, WORKFLOW_COMPLETED, CONFLICT_DETECTED, CONFLICT_RESOLVED, QUALITY_CHECK, HUMAN_APPROVAL, ERROR, CONFIG_CHANGED

- `log(entry: AuditEntry)` — persists via PersistenceLayer.appendAuditLog(), auto-sets timestamp if missing
- `query(filter: { eventType?, agentId?, taskId?, workflowId?, startTime?, endTime? }): AuditEntry[]`
- `getTaskTimeline(taskId: string): AuditEntry[]` — all events for a task, sorted by time
- `getAgentActivity(agentId: string, timeRange?: { start: Date, end: Date }): AuditEntry[]`
- `exportCSV(filter?): string` — generates CSV string with headers: timestamp, eventType, agentId, taskId, workflowId, data

## 3. src/monitoring/dashboard.ts — CLI Dashboard

Implement `Dashboard` class:
- Constructor takes MetricsCollector, AgentRegistry, TaskManager
- `render(): string` — returns a formatted ASCII string showing:
  - Header: "COWORK ORCHESTRATOR" + status (Running/Stopped) + uptime
  - Agents table: Name | Status | Load | Tasks Done (use box-drawing chars ┌─┐│└─┘)
  - Task Queue: Pending by priority (Critical: N, High: N, etc.), Running: N
  - Recent Activity: last 5 completed tasks with duration
  - Errors: last 5 errors
  - API Usage: total requests, tokens in/out
  - Use ANSI color codes: green for success, red for errors, yellow for warnings, cyan for headers
- `startLive(interval?: number)` — setInterval that does console.clear() + console.log(render()), default 2s
- `stopLive()` — clearInterval

Run `npx tsc --noEmit`. Fix any issues.
```
