## src/resilience/retry-strategies.ts

Implement retry and circuit breaker patterns:

### RetryStrategy interface
interface RetryStrategy {
  shouldRetry(error: Error, attempt: number): boolean;
  getDelay(attempt: number): number;
}

### ExponentialBackoff class implements RetryStrategy
- Config: { baseDelay: 1000, maxDelay: 60000, factor: 2, jitter: true, retryableErrors?: string[] }
- `shouldRetry(error, attempt)`: returns true if attempt < maxAttempts AND (error is in retryableErrors list OR error has status 429/500/503)
- `getDelay(attempt)`: calculates `min(baseDelay * factor^attempt, maxDelay)`, adds random jitter (0-30% of delay) if jitter=true

### CircuitBreaker class
States: CLOSED (normal), OPEN (failing — reject all calls), HALF_OPEN (testing recovery)

Config: { failureThreshold: 5, recoveryTimeout: 30000, successThreshold: 2, windowMs: 60000 }

State transitions:
- CLOSED → OPEN: when failure count >= failureThreshold within windowMs
- OPEN → HALF_OPEN: automatically after recoveryTimeout ms
- HALF_OPEN → CLOSED: after successThreshold consecutive successes
- HALF_OPEN → OPEN: on any single failure

Methods:
- `execute<T>(fn: () => Promise<T>): Promise<T>`
  - CLOSED: run fn, track success/failure
  - OPEN: immediately throw CircuitOpenError without calling fn
  - HALF_OPEN: run fn, on success increment success counter, on failure go back to OPEN
- `getState(): 'CLOSED' | 'OPEN' | 'HALF_OPEN'`
- `getStats(): { state, failureCount, successCount, lastFailure, lastStateChange }`
- `reset()` — force back to CLOSED, clear counters

### withRetry<T> utility function
async function withRetry<T>(fn: () => Promise<T>, strategy: RetryStrategy, maxAttempts: number): Promise<T>
- Calls fn up to maxAttempts times
- On error: check strategy.shouldRetry(). If yes, wait strategy.getDelay() ms, retry. If no, throw.
- Returns the first successful result
